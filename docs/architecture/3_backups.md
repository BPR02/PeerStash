# Backups: Restic & Retention Logic

PeerStash uses **restic** for all backup operations, ensuring that data is encrypted at the source before it ever leaves your NAS.

### Deduplication and Encryption
* **Efficient Storage:** Restic uses content-addressable storage, meaning if multiple files (or parts of files) are identical, they are only stored once on the destination peer.
* **Zero-Trust:** Every backup is encrypted with a unique repository password generated during `peerstash setup`. The receiving peer provides the storage space but cannot read the contents of your backups.
* **Deterministic Recovery:** The restic repository password is generated by hashing the username (as the salt) and password (as the original text). This means in the event of total file system corruption, the data can be restored by reinstalling PeerStash and using the same username and password.

### Retention Policy Syntax
PeerStash uses a specific string format for retention (e.g., `4w3d`), which maps to restic's `forget` policies:
* **`y` (Yearly):** Number of yearly snapshots to keep.
* **`m` (Monthly):** Number of monthly snapshots to keep.
* **`w` (Weekly):** Number of weekly snapshots to keep.
* **`d` (Daily):** Number of daily snapshots to keep.
* **`h` (Hourly):** Number of hourly snapshots to keep.
* **`r` (Recent):** Number of most recent snapshots to keep regardless of time.

Example: `1y2m4w` would keep 1 snapshot per year for the last year, 1 per month for the last 2 months, and 1 per week for the last 4 weeks.
